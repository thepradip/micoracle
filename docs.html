Voice commands for Claude Code, Codex CLI, OpenCode, and any focused terminal.

micoracle listens continuously, detects wake words, transcribes speech, cleans the prompt, pastes it into the locked target app, presses Enter, and can speak short status cues. It is local-first, cross-platform, and built around swappable STT and TTS backends.

Local by default macOS, Linux, Windows Swappable backends

Features

Continuous listening

WebRTC VAD, 30 ms frames, and a 300 ms preroll buffer keep wake words from being clipped.

Wake-word gate

Supports "Claude" and "Codex" with fuzzy matching for common speech-to-text mishears.

Two-step follow-up

Say the wake word alone, hear "listening", then speak the command within 8 seconds.

Noise filtering

Common silence hallucinations such as "thank you" and repeated phrases are silently dropped.

Platform adapters

macOS AppleScript, Linux xdotool/wtype, and Windows pywin32/pyautogui dispatch paths.

Clipboard-conscious paste

Clipboard-based dispatch paths restore the previous clipboard content after the prompt is sent.

Architecture

The runtime is a small producer-consumer pipeline. Capture and segmentation stay close to the microphone, while transcription and dispatch run in the worker thread.

Module overview

cd micoracle
pip install -r requirements.txt

Quickstart

# Focus Claude Code, Codex CLI, OpenCode, or another target terminal first.
./run_hands_free.sh

# Windows:
run_hands_free.bat

One-shot: say Codex, write a Python hello world. micoracle transcribes the command, pastes it, and presses Enter.

Two-step: say Codex, wait for listening, then speak the prompt within 8 seconds.

# Override backend selection
./run_hands_free.sh --stt-backend openai --tts-backend openai

# Pin a target app, required for Wayland
./run_hands_free.sh --target-app gnome-terminal

Troubleshooting

No input devices shown

Grant microphone permission to the terminal. On macOS, check Privacy & Security -> Microphone. On Linux, verify PulseAudio or PipeWire. On Windows, check Settings -> Privacy -> Microphone.

Wake word never fires

Run --list-devices, select the correct microphone, and say "Codex" or "Claude" clearly. Fuzzy matching covers many mishears, but low gain can hide initial consonants.

Wayland dispatch errors

Wayland blocks programmatic focus in many compositors. Use --target-app <name> and keep the target window focused manually.

macOS keystrokes ignored

Grant Accessibility and Automation permissions to the terminal application running micoracle.

Privacy & Security

  • Local backends stay on-device. MLX Whisper and faster-whisper do not upload audio for inference.
  • Cloud backends upload audio. OpenAI and Azure require sending captured speech to their services.
  • Clipboard is restored. micoracle may temporarily overwrite the clipboard on clipboard-based dispatch paths, then restores previous content.
  • No telemetry. The project does not include analytics or phone-home behavior.
  • Review permissions. Accessibility and automation permissions allow the app to type into focused windows.

Future Scope

  • Additional STT backends: ElevenLabs Scribe and Google Gemini are candidates for future cloud transcription support.
  • Stronger Linux target locking: improve X11 and Wayland dispatch so Linux can more closely match macOS and Windows target reactivation.
  • Packaged installers: provide smoother setup with platform-specific dependency checks.
  • Tray / menu bar control: add quick controls for pause, resume, backend selection, and target status.
  • Custom wake words: allow users to configure wake words beyond Claude and Codex.
